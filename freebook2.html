Freebook

An online Python/Flask web application for searching, saving and reading free e-books.

This was my final project for Harvard University's CS50x course on Edx.

After an introduction into using Python and Flask, making a project that worked with APIs and databases, we were asked to design and make our own project using the technologies we'd worked with on the course.

Freebook is an application where the user can make an account, search for free ebooks by author, publisher or title, and from those results they can catalogue books that they would like to read or have already read, or want to start reading immediately.

The books are then organised into 'My Bookcase', where they can be ammended or removed.

Login/Registration

  • The application automatically diverts to Login/Register form if user is logged out.
  • Both Login and Registration forms require all inputs to be filled and for usernames to be valid before submission, this is first checked asynchronously with jQuery and then returns a 500 code error through Python if the user has deactivated javascript.
  • Upon successful registration, a hash of the user's password is stored in the database.
  • All database handling is sanatised for security.

Homepage

  • Designed to guide the user to all main areas of the application.
  • A CSS-styled bookshelf contains all of the users' saved books and acts as a quick-look for the 'My Bookcase' page.
  • Underneath is a section that reminds the user of a book they're currently reading, this links to their 'Reading' page.
  • If the user has not catalogued any books, the shelf appears empty with a message suggesting they search for one, if the user isn't reading anything yet, the text in the lower section also suggests they search for a book.
  • Each book on the bookshelf, when selected, produces a modal with information about the book (title, authors, publisher, published date and description) and a button to close the modal.

Search

  • Using Google Books API.
  • Initally activated from the navigation bar for easy, instant access.
  • Within the search results page, the user can refine their search with a select form (options are 'author', 'publisher, 'title')
  • An input box is provided for a refined search, if not used, the refined search is carried out on the original query that was sent via the navigation bar.
  • A search produces 20 results, each with an image of the book, title, author(s), published date and a description if available.
  • Each result includes an option to record whether the user wants to read the book or has read the book already. This information is stored in two databases (for either wanting to read or having finished a book) and is found and ammended in 'My Bookcase' page and as a quick-look on the homepage's bookshelf too.
  • On selecting any of these options, a message is flashed asynchronously at the top of the page to inform the user of success or failure (eg. a book may already be in the user's finished list when they want to select that they've read it).
  • The image of the book is a link to start reading immediately, the user is redirected to their Reading page.

Reading page

  • The Reading page is accessed either through the navigation bar or from search results.
  • The books that a user is reading are listed vertically on the left of the page, and a Google Books viewer fills the rest of the page. When a book is selected it is displayed in the Google Books viewer.
  • Whilst reading a book, javascript records the page number the user is on and sends this information to update the database.
  • When accessed from search results, the new book is added to a database of what the user is reading with no page number (as the user has just started the book), the book is opened on the first relevant page as dictated by the Google Books API and viewer (eg the contents page).
  • When accessed from the navigation bar, the viewer displays the first book on the list at the page number the user was on during their last session.

My Bookcase

  • Books are displayed as image thumbnails in horizontal scrollable lists under two headings; 'To Read' and 'Books you've read', they work as a wishlist and a finished list.
  • Each thumbnail activates a modal of information about the book (title, authors, publisher, published date and description)
  • Under 'To Read' list, the modals give an option to append a book to their finished list, remove the book or close the modal. If they have finished the book, it is deleted from their wish list and is added to their finished list.
  • Under the finished list, the modals give an option to remove the book or close the modal.
  • On removing a book, the page is rerendered with a flashed message informing the user that the book was removed.

Settings

  • Options to change password, or permenantly delete account.
  • The form to change password operates as the login page does, with jQuery asynchronously checking the passwords adn a 500 code error thrown without javascript activate.
  • On deletion of the account, all data in the databases that include the user's id are deleted and the user is redirected to login/registration page with a flash message of successful deletion.

Freebook was made using the Google Books API, Bootstrap, SQLite, ajax, jQuery, Flask and Python.
